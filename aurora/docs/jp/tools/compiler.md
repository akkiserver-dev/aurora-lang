# コンパイラとCLIツールチェーン

Auroraコマンドラインインターフェース（`aurora`）は、ソースコードの実行、コンパイル、およびデバッグのためのツールを提供します。

## CLI コマンド

### 1. `run`
ソースファイルを実行します。`.ar` ファイルの場合はメモリ内でコンパイルして実行し、`.arobj` バイナリの場合は直接ロードして実行します。

```bash
aurora run main.ar [オプション]
aurora run main.arobj [オプション]
```

**オプション:**
- `--outputAst`: プログラムを解析し、抽象構文木 (AST) をコンソールに出力します。実行はされません。
- `-v`, `--verbose`: 詳細なログを出力し、エラー発生時にスタックトレースを表示します。

### 2. `compile`
ソースファイルを、実行可能なバイナリ形式 (`.arobj`) にコンパイルします。

```bash
aurora compile source.ar -o output.arobj
```

**オプション:**
- `-o`, `--output <path>`: 出力先のパスを指定します。指定しない場合は `<ファイル名>.arobj` として出力されます。
- `-v`, `--verbose`: コンパイル過程の詳細を出力します。

## オブジェクトファイル形式 (`.arobj`)

コンパイル済みのAuroraファイルは、以下の構造を持つバイナリドキュメントです：

1.  **マジックナンバー**: `0x4155524F` (ASCII 'AURO')
2.  **バージョン**: `0x00000002` (現在のVMバージョン)
3.  **定数プール**: 文字列、数値、関数、クラスなどのリテラルを効率的にロードするためのテーブル。
4.  **バイトコード**: 仮想マシン命令 (`OpCode`) のシーケンス。

## 仮想マシン (VM)
AuroraのVMはスタックマシン形式で動作します。**4バイトの命令**を使用し、`ArObject` インスタンスのスタックを操作します。Javaで実装されているため、JVMのメモリ管理や最適化の恩恵を受けることができます。
