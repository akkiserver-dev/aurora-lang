# ベストプラクティス

クリーンで安全、かつ効率的なAuroraコードを書くためのガイドラインです。

## Null安全性

1.  **非Null型を優先する**: 値が本当にオプションである場合のみ、Null許容型（`?`）を使用してください。
2.  **安全呼び出しを活用する**: 自力で null チェックを行う代わりに `?.` を使用してください。
3.  **デフォルト値を提供する**: エルビス演算子 `?:` を使用して、null の場合を適切に処理してください。
4.  **`!!` を避ける**: 他の手段がなく、かつ値が確実に null でないことが保証されている場合を除き、非null表明の使用は控えてください。

```ar
// 良い例
val name = person?.name ?: "ゲスト"

// 避けるべき例
val name = person!!.name
```

## エラーハンドリング

1.  **ビジネスロジックには `Result` を**: ユーザー入力エラーやネットワークタイムアウトなど、プログラムの通常の流れの一部として想定されるエラーには `Result<T, E>` を使用してください。
2.  **致命的なエラーには例外を**: 配列の範囲外アクセスなど、正しいプログラムでは発生すべきではない状況には例外を使用してください。
3.  **`?` で繋げる**: `?` 演算子を活用して、エラー伝播を簡潔に記述してください。

## 並行処理

1.  **I/O処理には Async**: システムの応答性を保つため、すべてのI/Oバウンドなタスクには `async`/`await` を使用してください。
2.  **計算処理には Thread**: 複数のCPUコアを活用するため、重い計算処理には `thread` を使用してください。
3.  **共有の可変状態を避ける**: スレッド間で可変状態を共有するよりも、`Channel` を介してデータをやり取りすることを優先してください。
4.  **Mutex の使用**: どうしても可変状態を共有する必要がある場合は、必ず `Mutex` で保護してください。

## コーディングスタイル

1.  **命名規則**: 標準の命名規則（関数・変数は `camelCase`、クラスは `PascalCase`）を遵守してください。
2.  **型推論**: 意味が明白な場合はコンパイラに型推論を任せ、パブリックAPI（公開関数など）には明示的に型を記述してください。
3.  **不変性**: 可能な限り `var` よりも `val` を優先してください。
